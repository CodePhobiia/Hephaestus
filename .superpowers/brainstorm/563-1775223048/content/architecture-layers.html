Foundation Platform: Layer Architecture

How ForgeBase fits into the Hephaestus source tree and how its internal layers interact

Layer Dependency Rules

domain/
Pure models, enums, value objects — zero imports from other layers
↓
repository/
Abstract contracts (ABCs) — imports only domain/
↓
store/sqlite/
aiosqlite impl
store/postgres/
asyncpg impl
store/blobs/
Local FS / S3
↓
service/
Business logic — uses repos + outbox
events/
Outbox + dispatcher + consumers
projection/
Branch view, diff, Git mirror
↓
factory.py
Bootstrap: reads config → wires backend → returns ready ForgeBase instance

Key Design Rules

domain/ has zero I/O
No database, no filesystem, no network. Pure Python dataclasses and enums.
repository/ defines contracts
Abstract base classes. Service layer and tests depend on these, never on store/.
store/ implements, never leaks
SQLite and Postgres implement the same ABCs. No backend-specific logic escapes this layer.
service/ owns transactions
State mutation + event emission happens here, in one transaction. Repos and outbox are injected.
